# 引き継ぎ書 — ゆうとの花だん

最終更新: 2026-05-11 / 現行バージョン: v1.0.1

---

## 概要

ちびっこのゆうとが、はたけに たねを まいて おはなを そだて、ママに はなたばを プレゼントする絵本風育成ゲーム。タイミングタップ式の「お世話」と、咲いた花の摘み取りでスコアを競う。

「育てる ゲージで タイミング を合わせる」「花を摘んで 集める」「ママに 渡して 喜ばれる」を1ラウンドで体験するワンプレイ完結型。難易度4段階(やさしい/ふつう/むつかしい/おに)で畑のマス数が指数関数的に増える(4 → 8 → 16 → 32)。

公開URL: https://windom21-cpu.github.io/k-games/yuto-garden/
QRコード: `yuto-garden/qr.png`

## 技術スタック

- 純粋な HTML5 + CSS + JavaScript (バニラ JS)
- 描画: **Canvas 2D Context** (960×540 固定、CSS で `max-width: 100%` で縮小スケール)
- 画面切替: DOM ベース(`.screen.active` クラスで切替)+ Canvas は `screen-game` / 終演CG時のみ描画
- 入力: **Pointer Events** で マウス/タッチ/ペンを統一処理。種まきは長押し(0.5秒)で配置 + 中央にピンクの進行リング描画、植え済みは即タップで撤去。育成中・練習中・摘み取り中は pointerdown 即時タップ判定
- 効果音: Web Audio API オシレータ(他ゲーム同方式)。`tone()` `sweep()` `chord()` ヘルパーで多種SFX生成
- 永続化: `localStorage["kgames-yuto-garden"]` に `highScores` (難易度別 TOP5) / `muted` / `tutorialShown` を保存
- ビルド工程なし。`index.html` を編集して push するだけで反映 (1〜2分)

## ゲーム仕様

### 画面遷移

```
タイトル(ゆうと+ママ)
  ↓
 [▶あそぶ] [📖そうさせつめい] [🏆ハイスコア]
  ↓
むずかしさ選択(やさしい / ふつう / むつかしい / おに)
  ↓
種まき画面
  ・空のますを <b>長押し(0.5秒)</b> → たね配置(進行リング表示)
  ・植えた たねは <b>タップ</b> で撤去
  ・1ますでも まいたら ▶せいちょうスタート 解放
  ↓
やりかた モーダル(▶ほんばん / ★れんしゅう / じゅんびちゅう)
  ↓                    ↓
本番(育成・お世話・摘み取り)   練習(3ラウンド・大きな単一畑)
  ↓                    ↓ 結果モーダル → ほんばん / もう1かい / 種まきへ戻る
全花が消化(摘めた/枯れた)
or ハート 0 で終了
  ↓
エンディングCG(ゆうと歩行 → ママに花束 → ママ笑顔 → スコア発表)
  ↓
[もういちど] [むずかしさをかえる] [タイトルへ]
```

### コアメカニクス

#### タイムゲージ

各 plot の下部に表示される横長バー。左から右へ進む。色:
- ピンク半透明: 進行済み部分
- **黄色帯 (sweet)**: タップで `good`(+20点・花が +1本)
- **赤色帯 (perfect, 黄帯の中央)**: タップで `perfect`(+50点・花が +2本)
- それ以外でタップ: `normal`(+5点・増殖なし)
- バー右端まで届くと: `miss`(-10点)

帯の中央位置 `sweetCenter` はゲージ毎に [0.35, 0.85] のランダム。**おに モードのみ** バウンドして移動(`moveSpeed = 0.28`/秒)。

#### 成長サイクル

各 plot は花の種類(`flower.cycles` = 2〜4)に応じて N 回のお世話サイクルを必要とする。1サイクル = 1 ゲージ通過(タップ or miss)。N 回終わると **bloom** 状態へ:
- `bloomAnimT` で開花アニメ(0.5秒)
- 自動で **摘み取りゲージ**(緑、4.5秒)が始動
- ゲージ消失前にタップで `pick` 成功 → 花の `baseScore × flowerCount` を加算
- 摘み損ね → wither(枯れ): -20点 + ハート -1

#### 花の種類

| ID | 名前 | 必要サイクル | baseScore | 描画特徴 |
|---|---|---|---|---|
| tampopo | タンポポ | 2 | 30 | 黄色の放射ライン |
| asagao | あさがお | 2 | 30 | 紫トランペット形 + 星模様 |
| tulip | チューリップ | 3 | 50 | 赤いチューリップカップ |
| cosmos | コスモス | 3 | 50 | 8枚のピンク楕円花弁 |
| sunflower | ひまわり | 4 | 80 | 14枚オレンジ花弁 + 茶色の中心 |
| rose | バラ | 4 | 80 | 同心円の深紅ローズ |

種をまく時点では花の種類は不明。`せいちょうスタート` 時点で各 plot にランダム割当(`FLOWERS[Math.random() * 6]`)。

#### 難易度

| 難易度 | マス数 | 配置 | ゲージ速度 | スイートスポット幅 | スポット移動 |
|---|---|---|---|---|---|
| やさしい | 4 | 2×2 | 0.10〜0.16/s (6.3〜10秒/本) | 34% / perfect 12% | 固定 |
| ふつう | 8 | 4×2 | 0.16〜0.25/s (4〜6.3秒) | 22% / perfect 7% | 固定 |
| むつかしい | 16 | 4×4 | 0.25〜0.38/s (2.6〜4秒) | 14% / perfect 4.5% | 固定 |
| おに | 32 | 8×4 | 0.38〜0.55/s (1.8〜2.6秒) | 11% / perfect 3.5% | **ランダム移動** |

#### 練習モード

`やりかた` モーダルで「★ れんしゅう してから」を選ぶと:
1. 種まき結果を `State._savedPlots` に退避
2. 画面中央に大きな単一 plot(320×240)を配置・タンポポ固定
3. 3ラウンドのタイムゲージタップ練習(クールダウン 0.9秒)
4. 結果を画面上部に大きく表示(`perfect ☆` / `good !` / `おせわ +5` / `ミス…`)
5. 終了後に結果モーダル → ほんばん継続 / 再挑戦 / 種まきへ戻る

#### スコアと終了条件

- **スコア**: お世話 (-10〜+50) + 摘み取り (baseScore × flowerCount)。最低 0 にクランプ
- **ハート**: 3個。摘み損ね(枯れ)で -1。0 でゲームオーバー(早期終了・sad ending)
- **終了**: 全 plot が done(picked or withered)になると happy ending、ハート 0 で gameover ending
- **isNewHi 判定**: ソート後の `list[0]` が今回 entry と同一参照ならハイスコア更新

#### 評価ランクなし

S/A/B/C のような評価は付けない。**スコア絶対値**のみで競う(難易度別 TOP5)。

### エンディングCG (drawEndingScene)

| 時間 | 演出 |
|---|---|
| 0.0〜1.0s | 暖色オーバーレイ(夕焼けトーン)が フェードイン |
| 1.2s | テキスト「おうちに かえって…」フェードイン(happy 時)/「あれ… おはなが…」(gameover 時) |
| 2.0〜5.0s | ゆうとが 画面外左 → 中央 (CANVAS_W * 0.50) へ歩行(`easeOutCubic`) |
| 3.5s | テキスト切替「ママに おはなを わたそう ♥」 / 「ママは ちょっと しょんぼり…」 |
| 5.5s〜 | 摘んだ花がある場合: ゆうとの腕を `extend` ステートに(花束を差し出す) |
| 6.0s〜 | ママが `happy` mood (笑顔・ほっぺピンク・浮遊ハート) / `sad` mood (gameover 時) |
| 6.5s〜 | テキスト「うふふ、ありがとう ゆうと ♥」 + スコアカード スライドアップ |
| 6.5s+ | ハイスコア更新時: 黄色バッジ ★ハイスコアこうしん★ + キラキラ SFX |

花束は `drawBouquet()` が `State.pickedFlowers` を参照して合成:
- 紙の包み(円錐形・リボン)を底に
- 摘んだ花を扇形に配置(レアな花ほど大きく)
- 数が多いほど span が広がる

ゲームオーバーでも 1本でも摘めていれば花束は表示される(arms-extend も発動)。

## 既知の制約 / 今後の余地

### 既知の制約

- **タイムゲージ速度のランダム性**: 同じ難易度でも各サイクルでランダム速度のため、パターン記憶しづらい(意図通りだが、極端に速いゲージが連続すると体感が悪い場合あり)
- **連打可能**: ゲージ範囲内ならどのタイミングでもタップが通るので、`スパムタップ → normal 多発` で進めることもできる(難易度低下の経路)
- **練習結果はスコアに反映されない**: 純粋なチュートリアルなので意図通り
- **花のレア度は均等抽選**: タンポポ(2サイクル・30点)とバラ(4サイクル・80点)が同確率。レア度別の出現重みは未実装
- **花束の合成は単純配置**: 摘んだ花を扇形に並べるだけで、種類別グルーピングや色合わせはしていない
- **スマホ縦持ち**: 横にデフォルトで遊べるが、縦持ちでは UI 圧縮(タイル小・ボタン小)。実用は可能

### 今後の余地

- **連続成功ボーナス**: 3〜5回連続 perfect で「コンボ」+α 倍率
- **時間制限モード**: タイムアタック(全花消化までの秒数で評価)
- **花束カスタマイズ**: 配置や包装紙の色を選べる
- **ママの反応バリエーション**: 摘んだ花の質によって 5段階くらいのリアクション
- **花の出現重み**: 難易度別にレア花の出現確率を調整(おに モードで rare 多めなど)
- **チャレンジ実績**: 「全 perfect クリア」「ノーミス」等のバッジ
- **タップ即効果のみ → 配置型ミニゲーム化**: 「水やり」「肥料」を別アクションに分けて2系統に
- **ステージエディタ / ステージ追加**: 現状はランダム生成のみ。固定マップ・お題ステージを追加する余地あり
