# 引き継ぎ書 — スターマイマイ

最終更新: 2026-07-19 / 現行バージョン: v1.2.0

---

## 概要

カタツムリ(マイマイ)を育てて1年間のレースを戦う育成シミュレーション。スターホース風の「育成 → 出走 → 引退 → 配合(繁殖)」サイクルを、マイマイ題材でコミカルに翻案したもの。

- サイコロ10個をふって生まれた1頭を、半月きざみの1年間(24ターン)で調教
- 年間12レース(G3→G2→G1)を任意に選んで出走し、賞金をかせぐ
- 最終目標は12月後半の最高峰レース **「マイマイカップ」** 制覇
- 引退した個体は **殿堂** に永久保存され、**配合(v1.1 で実装済み)** の親としていつでも使える
- 牧場(スロット)単位で、レースごとのレコードタイムと通算勝利回数を蓄積(v1.2)

公開URL: https://windom21-cpu.github.io/k-games/star-maimai/
QRコード: `star-maimai/qr.png`

## 技術スタック

- 純粋な HTML5 + CSS + JavaScript (バニラ JS、依存ゼロ)
- 描画:
  - 画面切替は DOM ベース(`.screen.active` クラス)+ モーダル(`.modal-back.active`)
  - タイトルアニメ / 厩舎のマイマイ / レース実況 / 引退式は **Canvas 2D**(`drawSnail()` が共通のカタツムリ描画関数。カラの値で殻が大きく・キラキラになる)
- 入力: click / タッチ(`touch-action: manipulation`)
- 効果音: Web Audio API オシレータ(`SFX` オブジェクト)。🔊/🔇 トグルは `localStorage["starMaimai.mute"]` に永続化
- 永続化: `localStorage["starMaimai.slot0..2"]`(下記セーブ仕様)
- ビルド工程なし。`index.html` を編集して push するだけで反映 (1〜2分)

### マイマイの能力(4ステ+隠し)

| 表示 | 意味 | 上限 | 育て方 |
|---|---|---|---|
| ぬめり | 速さ(基本走力) | 120 | 調教「ガラス板すべり」 |
| 体力 | スタミナ。長距離でバテにくい | 120 | 調教「ながい散歩」 |
| 根性 | 残り24%のスパート+バテ後のねばり | 120 | 調教「タカとにらめっこ」 |
| カラ | カラカラ(乾いた)地面への耐性 | 120 | **調教では伸びない。カルシウムのエサのみ** |

ランク表示: F(<22) E D C B A S(≥110)。

**隠しステータス(画面に出さない)**:
- `hidden.gf` — 成長係数。サイコロ合計(10〜60)から 0.90〜1.25
- `hidden.apt.{num,sta,gut,shl}` — **調教効果**。ステ別に gauss(1.0, 0.25) を 0.6〜1.7 にクランプ。調教の伸び = 基本値 × apt × gf × 疲労補正。**サイコロが悪くても apt が高ければ大化けする**(ユーザー要望)
- ヒントは (1) 誕生時のコーチコメント(apt 平均×gf の水準で5段階の台詞)、(2) 調教時の「ものすごい伸びだ!」等の実感コメントのみ

### 誕生(サイコロ10個)

- 青3個→ぬめり(合計×3.0)、緑3個→体力(×3.0)、橙2個→根性(×3.5)、紫2個→カラ(×3.5)
- 合計値が成長係数 gf にも影響(=良い目は初期値と成長の二重取り)
- 名まえは手入力(最大8文字)or「おまかせ名まえ」生成

### ターン(半月×24)

毎ターン、**エサを選んで行動1つ**:
- エサ: ふつう(無料) / カルシウム(200G, カラ+2) / 特選カルシウム(500G, カラ+4・体力+1)。資金不足時は自動でふつうに降格
- 行動: 調教3種 / 休養(つかれ-50, 調子上向き) / レース出走(開催ターンのみ)
- つかれ: 調教+17前後、レース+22、80以上で調教失敗35%。55以上でレースの速度ペナルティ
- 調子: 5段階(ぜつ不調〜ぜっ好調)。毎ターン確率でゆらぐ。レース速度 ×0.90〜×1.09

### 天候・地面(マイマイの本領)

月別の雨確率(6月高・7〜8月低)+ レース毎の `wetBias` で抽選:
- **ぬれぬれ**(☔): 全員 ×1.09。カラ無関係 → 実力勝負・番狂わせの舞台
- **しっとり**(☁): ほぼ等倍
- **カラカラ**(☀): 速度 ×0.76〜1.00(カラ0→120で線形)+ スタミナ消費 ×1.3(カラで軽減)。**カラを鍛えた個体だけが晴れでも強い**(ユーザー要望どおり)

### レースシミュレーション

`stepRace()` を dt きざみで回す物理風モデル(スキップ時は `fastForwardRace()` で 0.1s きざみ一括計算 — 実況と同一ロジック):
- 速度 = (2.1 + ぬめり×0.03) × 地面係数 × 調子係数 × 疲労係数 × ゆらぎノイズ
- スタミナ = 55 + 体力×2.1。毎秒 6.5 消費(カラカラで増、スパート中×1.35)。切れると速度×(0.52+根性×0.0013)=バテ
- 残り24%でスパート: ×(1 + 根性×0.002×乱数)
- スタートに 0〜0.7s の出遅れ乱数
- 実況: ×1/×4倍速トグル、結果へスキップ、ぬめり跡・バテ表示・🥇🥈🥉

### 引退・殿堂

- 24ターン経過で自動引退。途中引退も可(確認モーダル、取消不可)
- 殿堂エントリに `canBreed: true` を必ず付与 — **引退個体は一生配合可能**(ユーザー方針)。`hidden`(gf/apt)もスナップショット保存され、遺伝計算に使われる
- 資金・殿堂・記録はスロット内で世代をまたいで引き継ぎ。引退→迎え方選択へ(年数+1)
- 引退式には「タイトルへもどる」ボタンあり(v1.1.1)

### 配合・世代交代(v1.1)

- 殿堂に1頭でもいれば、新しいマイマイを **配合** でつくれる(野生スタートも選べる)
- 親選び: 殿堂の `canBreed` 個体+「🌿野生のマイマイ」カード(画面に入るたびに能力を引き直し。ステは `wildStatMin..Max`=25〜70、apt は新規 gauss)から **2頭をタップ選択**。3頭目をタップすると古い選択が外れる
- 遺伝(`makeSnailFromDice(dice, name, breed)` / パラメータは `TUNE.breed`):
  - **初期ステ加算** = 両親平均ステ × `statInheritRate`(0.15) × 変異 `gauss(1, 0.35)` を 0.2〜1.9 にクランプ → サイコロ由来値に上乗せ(「両親ゆずりのちから」としてサイコロ画面に表示)
  - **apt(隠し調教効果)** = 両親平均を 25% だけ 1.0 へ回帰させた値を中心に `gauss(±0.18)` → 強い血統は維持されつつインフレしない。**変異幅広め=弱親から大物が出る余地**(ユーザー指定)
  - **gf(成長係数)** = サイコロ由来×0.55 + 両親平均×0.45 + `gauss(0, 0.03)`(0.85〜1.32)
  - 世代 `gen` = max(親の gen)+1(野生親は gen 0)、`parents` に両親名を保存
- 殿堂・厩舎に世代バッジ(第n世代)と両親名を表示

### レコード・通算勝利数(v1.2)

- スロット単位で `records[raceId] = {time, name, year, ground}`(自己ベスト)と `raceWins[raceId]`(通算勝利数)を蓄積
- `applyRaceResult()` で更新。既存レコードを塗り替えると結果画面+日誌に「🏅 コースレコード更新!」(初出走は黙って基準を記録)
- 表示は 📅「レースと記録」モーダル下部の「🏅 これまでの記録」— **出走したレースだけ** 並び、タイム+地面タグ+保持マイマイ+樹立年を表示

## セーブ仕様(重要)

`localStorage["starMaimai.slot0" | "slot1" | "slot2"]` に JSON:

```
{ version: 3,            ← SAVE_VERSION
  money, year,
  hall: [殿堂エントリ...],           ← 各エントリに gen / parents / canBreed
  current: マイマイ|null,            ← gen / parents を含む
  records:  { raceId: {time, name, year, ground} },  ← レース自己ベスト (v3)
  raceWins: { raceId: 勝利回数 },                     ← レース通算勝利数 (v3)
  updatedAt }
```

**マイグレーション**: `MIGRATIONS = { 旧version: (data)=>新data }` のチェーン方式。`loadSlot()` が読み込み時に `migrateSave()` を通し、変換後は即再保存する。

- `MIGRATIONS[1]`(v1→v2): current / hall の全個体に `gen: 1` `parents: null` を付与
- `MIGRATIONS[2]`(v2→v3): `records: {}` を追加し、`raceWins` は殿堂+現役の `results` の1着から復元(タイムは過去に保存していないため復元不可、以後の出走から記録)

> ⚠ **セーブ構造を変えるときの手順**: (1) `SAVE_VERSION` を +1、(2) `MIGRATIONS[旧番号]` に変換関数を追加。これを守れば公開後のアップデートでプレイヤーのデータが消えない。

## バランス設計と検証

調整パラメータは **`TUNE` オブジェクトに全集約**(初期値倍率・成長・疲労・エサ・レース物理・NPC強度以外に `GRADE_NPC` / `GRADES` / `RACES` / `MONTH_WET`)。

v1.0.0 決定値は、実コードをヘッドレスで 500シーズン×3方針 自動プレイして調整した:

| プレイスタイル | G3勝率 | G2勝率 | G1勝率 | MC制覇/シーズン |
|---|---|---|---|---|
| 雑(ランダム調教・カルシウムなし) | 63% | 29% | 5% | 1.4% |
| 丁寧(最低ステ調教+カルシウム) | 65% | 39% | 9% | 3.6% |
| 上級(ぬめり体力集中+カルシウム) | 73% | 55% | 22% | 19% |

設計意図 =「初代でも運が良ければ勝てる。うまく育てれば数世代で届く」(ユーザー指定)。

**世代交代バランス(v1.1.0 決定値、400試行×4世代)** — 親は殿堂のベスト2(足りなければ+野生):

| プレイスタイル | 第1世代 | 第2世代 | 第3世代 | 第4世代 |
|---|---|---|---|---|
| 上級(ぬめり体力集中+カルシウム) MC制覇/シーズン | 22% | 55% | 64% | 68% |
| 丁寧(最低ステ調教+カルシウム) MC制覇/シーズン | 1.5% | 8.5% | 13% | 17% |

**「レースに出ず調教だけした個体を種親にする」搾取の検証済み(2026-07-19)**: 引きこもり調教でステ合計310〜423の親は作れるが、遺伝率15%で子への上乗せは僅か+5pt程度。一方でそのシーズンの賞金・タイトルを丸ごと失うため、**2シーズン合計では出走し続ける方が明確に得** → 対策不要と判断(理由: 遺伝率15%の希釈、apt/gf は調教で盛れない、ステ上限120、賞金喪失)。

テストハーネスはセッションの scratchpad にあった `simtest.js` / `flowtest.js` / `balance.js`(DOM スタブ+vm で index.html の `<script>` を実行し、`doTrain`/`setupRace`/`applyRaceResult` など実関数で回す方式)。再調整時は同方式を再作成すると早い。UI 確認はヘッドレス Chrome(`--headless=new --screenshot`)+検証用データ注入 HTML で行った。
