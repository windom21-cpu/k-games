# ポピンズ 引き継ぎ書

最終更新: 2026-04-30（ボーナス面追加・スキン10種・タッチ操作改善・各種iOSバグ修正）

公開URL:
- ゲーム本体: https://windom21-cpu.github.io/dtt-mini/popins.html
- QRコード（タブレット用）: 同リポジトリの `qr-popins.png`

## 概要

セガのメダリンクシリーズ「マジカルポピンズ」を参考にした、横スクロール型ブロック崩し。
傘（パラソル）でボールを跳ね返してブロックを崩しながらコインを稼ぎ、ワールドマップを巡る。

- 元ネタ参考: https://medalink.sega.jp/game_m_in_p.html
- 構造: 単一 HTML 自己完結（外部リソース／サーバー不要）

## ファイル

- `popins.html` … ゲーム本体。PC／タブレット両対応の単一ファイル
- `qr-popins.png` … 公開URLのQR（600×600）

## 起動方法

- PC: `popins.html` をダブルクリック → ブラウザで開く
- タブレット/スマホ: QRコードを読み取り → Safari/Chromeで開く（横向き推奨）

## 操作方法

### PC
- **マウス移動**: 傘を左右に動かす
- **左クリック / Space**: 球を発射（球残数を1消費）
- **R**: ステージリセット＋球+10（デバッグ用）
- **🔊/🔇**: 音のON/OFF

### タッチ（タブレット/スマホ）
- **画面どこでもタッチ**: 傘がX位置に追従
- **タップ（ほぼ動かさず離す）**: 球発射
- **マップ画面のSTART**: タップで開始
- 専用スライドバーは廃止 → **画面全体が操作領域**

## ゲーム仕様

### マップ構造（6ノード×複数巡）
```
1巡目: 春🌸 → 夏☀️ → 💰ボーナス → 秋🍁 → 冬❄️ → 銀ボス🥈
2巡目: 春   → 夏   → 💰ボーナス → 秋   → 冬   → 金ボス🥇 (GAME COMPLETE!)
3巡目以降: 銀/金 が交互に現れる無限ループ
```

- 5番目（idx=5）がボスノード。`loopCount` の偶奇で銀/金が動的に決まる（`currentBossType()`）
- 3番目（idx=2）がボーナスノード。コイン専用ステージ
- `nodeAt(idx)` ヘルパーで季節 / ボス / ボーナスを統一的に解決

### ノード定数
| 定数 | 値 | 意味 |
|---|---|---|
| `TOTAL_NODES` | 6 | マップノード総数 |
| `BONUS_IDX` | 2 | ボーナス面のインデックス |
| `BOSS_IDX` | 5 | ボス面のインデックス |

### コアループ
- マップで `▶ START` をタップしてステージ開始
- ブロック / ザコ / コインが右からスクロール
- 全敵を撃破 or 流し切ってクリアバナー → マップで次のノードへ歩く
- ボスステージはスクロール無し、ボス1体と対峙

### コイン経済（リソース管理型）
- **球残数（🎱）** = プレイ可能リソース。発射で1消費
- **コイン（💰）** = スコア兼継続コスト
- 初期: 球10、コイン0
- 球切れで「100コイン → 球+10」継続ポップアップ
- 100コイン未満 or 「ゲーム終了」選択で **GAME OVER**

### 報酬
| 行動 | コイン |
|---|---|
| ブロック破壊 | 1 × max(1, コンボ) |
| コインタイル破壊（ボーナス面） | 2 × max(1, コンボ) |
| ザコ撃破 | 5 × max(1, コンボ) |
| 通常ステージクリア | +30 |
| **ボーナス面クリア** | **+100** |
| 銀ボス撃破 | +500（じゃらじゃら35枚） |
| 金ボス撃破 | +1000（じゃらじゃら70枚＋GAME COMPLETE） |

### コンボ
- 球が傘に戻るまでに連続で当てた数
- 傘でバウンドするとリセット
- 倍率は `max(1, combo)`（コンボ5なら5倍）

### ステージ構成
- 通常ステージ ≈ **30秒**、`STAGE_DURATION = 30`
- ボーナスステージ ≈ **15秒**（半分）、`BONUS_DURATION = 15`
- スクロール速度 60px/s
- `currentStageLength()` 関数で動的にステージ長を返す
- 通常チャンク：9種からランダム供給（バンド／階段／散布／市松／アーチ／砂時計／ザコ要塞／ザコ列／隙間付きバンド）
- ボーナスチャンク：14列×9行のコイン敷き詰め（`chunkCoinDense`）
- ザコ HP 3、撃破で5コイン
- ボス HP: 銀30 / 金60。攻撃なし。**5〜7秒ごとに2〜3秒の防御モード**（赤い怒り顔＋シールドで無敵）

### コインタイル（ボーナス専用）
- ブロックの `bonus: true` フラグで識別
- 描画: 金色の円＋$記号＋放射グラデーション（コイン風）
- 当たり判定はAABB（ブロックと同じ円vs矩形）
- 効果音は `SE.item()`（チャイム）
- アイテムドロップなし

### アイテム（出現は各面1個程度）
| | 効果 | 持続 |
|---|---|---|
| **W (Wide)** | 傘の幅×1.6 | 10秒、複数取得で時間加算 |
| **M (Multi)** | その場で球を+2個生成 | 一回限り（生成された球は通常通り） |

- ドロップ率: ブロック0.6%、ザコ6%、コインタイル0%
- ブロックの場所に出現、左に流れて消えるまで有効
- 球が当たると発動、球は通過（跳ね返らない）

### 傘（パラソル）
- 半幅 `halfW = 65`（直径130px）→ 指で隠れにくい大きめサイズ
- 配置 y = H - 50（画面下端から50px上）
- ドーム上半円のみ反射面（持ち手や下半分は素通り）= 視覚と判定が一致
- 当たる位置で反射角が変わる（中央=真上、端=斜め60°、常に上向き）

### 傘スキン（10種、ステージクリアで1つずつ解放）
| ID | 名前 | パターン |
|---|---|---|
| 0 | クラシック | 赤＋黄縁（初期） |
| 1 | オーシャン | 青 |
| 2 | チェリー | ピンク |
| 3 | サンシャイン | 黄＋オレンジ縁 |
| 4 | フォレスト | 緑 |
| 5 | ロイヤル | 紫＋金縁 |
| 6 | ストライプ | 赤白縞 |
| 7 | ドット | 白地に赤水玉 |
| 8 | レインボー | 七色グラデ |
| 9 | ステルス | 黒＋銀縁 |

- 描画: `fillSkinDome(R, skin)` ヘルパーで4パターン（solid/stripes/dots/gradient）対応
- ボス撃破でも解放対象。10種すべてで10ステージ分
- タイトル画面の **「🌂 傘を選ぶ」** セクションでタップ選択
- ロック中は 🔒 表示、選択中は黄色枠
- グリッドは **5列×2行固定**（CSS Grid）でスマホ縦圧縮しても崩れない
- localStorage `popins-skin` (選択ID), `popins-skins-unlocked` (解放配列)

## ワールドマップ

- 6ノードがジグザグ配置で表示
- マップタイトル: `～ WORLD MAP ～ N巡目`
- 傘キャラがクリア済みノードから次ノードへ**ぴょんぴょん歩く**
- 到着後、現在ノードに **▶ STARTボタン**（パルスアニメ）→ タップで該当ステージ開始
- 起動時の初回はマップ外左から春ノードへ歩いてくる

## ビューポートフィット（W動的化）

- 論理 W は固定でなく、**ビューポートのアスペクトに合わせて伸縮**
- `let W = 960` 初期値、`fitCanvas()` 内で `W = clamp(H * aspect, W_MIN=720, W_MAX=1920)` 再計算
- 横長ウィンドウ → ゲーム領域も横長に。左右の黒帯が消える
- H は固定 540
- リサイズ時は umbrella.x も画面内にクランプ
- nextChunkX、ボス位置、マップ位置すべて live W を参照するので追従する

## オーディオ

### 効果音 (SE) — Web Audio API 自前合成
- 発射 / 傘バウンス / ブロックヒット / ブロック破壊 / ザコ撃破
- アイテム取得 / コインタイル取得 / ボスヒット / ボス防御で弾かれ / ボス撃破ファンファーレ
- ステージクリア / 球切れ / ゲームオーバー

### BGM — Web Audio API 自前合成（季節別ループ）
- 春: のんびり (92 BPM, C-G-Am-F)
- 夏: 元気 (132 BPM, 跳ねるベース)
- **ボーナス: 陽気で速い (168 BPM, 8拍ループ)** ← 新規
- 秋: 物悲しい (84 BPM, Aマイナー)
- 冬: 静寂 (76 BPM, 鈴音風)
- 銀ボス: 緊迫 (138 BPM, Eマイナー)
- 金ボス: 荘厳 (156 BPM, Cマイナー、3層)

各曲は4小節（16拍）ループ、ボーナスのみ2小節（8拍）。`updateBgm()` が0.5秒先まで先読みスケジュール。
ステージ開始でフェードイン、クリア／ボス撃破でフェードアウト。

### iOS Safari 対応
- AudioContext は初回ユーザー操作で `ensureAudio()` 経由で起動
- `interrupted / suspended` どちらの state でも `resume()` を試行
- **無音1サンプルバッファを再生**してロック解除（iOS 必須テクニック）
- `pointerdown / touchstart / touchend / mousedown / click / keydown` 全部にリスナー
- `startStage` / `launchBall` の冒頭でも明示的に `ensureAudio()`

### BGM 切替（既知バグの修正）
- 旧: `stopBgm` のクリーンアップが共有 `bgmActiveOscs` を全部停止 → 新BGMの音まで止めていた（春以降が無音）
- 新: 停止時にスナップショット → スナップショット内のオシレータだけ stop。新BGMは新配列に積む

### ミュート
- トップバーの 🔊/🔇 ボタン
- localStorage `popins-muted` に保存（次回起動で復元）

## セーブ / ハイスコア

### セーブ
- localStorage `popins-save` (v1)
- セーブタイミング: マップ歩行が完了して `MAP_WAIT` に入る瞬間（次のステージ開始前）
- 保存内容: コイン / 球 / スコア / 最大コンボ / ステージ番号 / 周回 / 現在ノード
- ゲームオーバーで自動クリア

### ハイスコア
- localStorage `popins-hiscores` (top 5)
- ゲームオーバー時にコイン保有数で順位付け
- ランクインで「🎉 NEW RECORD!」表示
- タイトル画面で常時表示

### タイトル画面
- 起動時に表示
- ハイスコア＋スキン選択＋始める/続きから
- 5列固定グリッドでスマホ縦圧縮にも対応
- `max-height: 92vh + overflow-y: auto` でスクロール可能（保険）
- `@media (max-height: 460px)` でさらに圧縮（iPhone 横向け）

## タッチ操作

### 直接操作モード（M16以降）
- 画面のどこでもタッチ → 傘が指のX位置に追従
- 旧バージョンの「下部スライドバー」は廃止
- ドラッグ判定は **CSS pixel** ベース（端末解像度に依存しない）
- `TAP_THRESHOLD_CSS = 16`、これ以下なら「タップ」扱いで発射
- マップ STARTボタンは座標チェック+16px のヒット余裕

### iOS バグ対策
- `body { touch-action: manipulation }`（none だと synthesized click が抑止される）
- モーダルボタンは `touchend / pointerup / click` 全てにバインド（`bindButton` ヘルパー）
- `touch-action: manipulation` でダブルタップズーム自動抑止
- `tryLockLandscape` は `pointerdown` ではなく `startStage` 起点（タイトルの始めるボタンを邪魔しない）
- 横画面ロック未対応端末向けに CSS の `#orientation-overlay`（縦向き＋pointer:coarse 時表示）

## フェーズステートマシン
- `PHASE_PLAYING` / `PHASE_CLEAR_BANNER` / `PHASE_MAP_TRANSIT` / `PHASE_MAP_WAIT`
- `paused` フラグで全更新を停止（タイトル／継続ポップアップ／ゲームオーバー画面表示中）

## 描画レイヤ順
1. 季節別グラデ背景
2. パーティクル
3. ブロック（通常 or コインタイル） → ザコ → ボス → アイテム → 傘
4. ステージ進行バー（ボス戦中は非表示）
5. コインレイン（ボス撃破時）
6. アクティブ効果インジケータ（WIDE残時間など）
7. ボール（**シルバーメタリック**＋外周グロー）
8. オーバーレイ（クリアバナー / マップ）

## 当たり判定
- 円(球) vs AABB(ブロック/ザコ/ボス/コインタイル): 最近接点での反射、めり込み時はAABB内部から最短経路で押し戻し
- 傘のドーム判定は半円: `dy <= b.r * 0.6 && dist <= R + b.r`
- 反射角は当たった位置から計算（中央=真上、端=斜め）

## 解像度
- 論理 W (動的) × H 540 / DPR 対応
- `fitCanvas()` でビューポートに合わせてアスペクト維持スケーリング
- safe-area-inset で iPhone ノッチ対応

## 既知の制限・今後の候補

- **マルチボール無制限**: 連発でボールが指数的に増える可能性あり
- **ボス無攻撃**: 仕様通りだが、長期的には弾発射などの追加で緊張感UP余地
- **ステージ難易度固定**: 周回数で敵強化などの仕組みなし
- **縦向き禁止**: 縦向きはオーバーレイで横向きを促す（A案で確定）
- **タイトル画面の演出**: アニメーション／キャラ立ち絵などで強化可能
- **ヘルプリンクなし**: DTTみたいなguide.html風の攻略ガイドがあると親切

## 重要なJS構造ポイント（次回触る人向け）

### TDZ注意
- `let W = 960` 直後に `fitCanvas()` を呼ぶが、`umbrella` はまだ未宣言
- `typeof umbrella` は **strict mode + let の TDZ で例外**になる
- `try { umbrella.x = clamp(...) } catch(_) {}` で初回呼び出しを許容
- 同じ理由で `umbrella` 参照前に typeof チェックは使わない

### bindButton ヘルパー
- 全モーダルボタンを `touchend / pointerup / click` にバインド
- 700ms の firing フラグで重複発火防止
- iOS Safari での synthesized click 抑止対策

### updateBgm の予約スケジュール
- `bgmNextScheduleAt` が `now + 0.5` を切ったら次の小節を予約
- 16/8拍ループのスケジュールが切れ目なく続くように設計
