# 引き継ぎ書 — メジェドを救え!

最終更新: 2026-05-06 / 現行バージョン: v1.1.1

---

## 概要

1991年に DMA Design が発表した **Lemmings** の簡略版。プレイヤーは固定地形上を右へ歩く「メジェド神」たちにアイテムを置いて経路を作り、規定数を **ゴールのピラミッド** へ導く誘導パズルアクション。

題材として **エジプト神話のメジェド神**(うっすら布をかぶり目だけ見える、白い・棒人間風)を採用。基本(A)に加えてファラオ・戦士・司祭・ミイラ・子供の6種を別 SVG として用意し、ステージ進行に応じて速度や挙動の異なる派生種が混入する。

公開URL: https://windom21-cpu.github.io/k-games/Lemmings/
QRコード: `Lemmings/qr.png`

⚠ ディレクトリ名は `Lemmings/`(先頭大文字)で他ゲーム(全て小文字)と不揃いだが、開発当初に作成した名前のまま運用している。改名する場合は `git mv` + 公開URL/QRコードの再焼成が必要。

## ファイル構成

```
k-games/Lemmings/
├─ index.html              ← ゲーム本体(これだけで動く)
├─ medjed_A_basic.svg      ← 基本のメジェド
├─ medjed_B_pharaoh.svg    ← ファラオ(青ネメス・金縞)
├─ medjed_C_warrior.svg    ← 戦士(槍持ち)
├─ medjed_D_priest.svg     ← 司祭(金リング・紫目)
├─ medjed_E_mummy.svg      ← ミイラ(包帯・赤目)
├─ medjed_F_child.svg      ← 子供(小柄・大きな目)
└─ qr.png                  ← 公開URLのQRコード
```

ゲーム本体は単一HTMLに同梱(CSS/JS インラインのバニラ JS)。SVG は外部参照で `Image()` ロード後にオフスクリーン Canvas へラスタライズして描画している。

---

## 技術スタック

- 純粋な HTML5 + CSS + JavaScript (バニラ JS)
- 描画: **Canvas 2D Context**(960×544 固定、CSS で `max-width: 100%` で縮小スケール)
- メジェドスプライト: SVG → `Image` → 28×36 px の Canvas にプリラスタライズ(右向き/左向きを別キャッシュ)
- タイル: 32×32 px、30列×17行 グリッド。タイル種別を整数定数 `T.*` で管理
- 入力: マウス click / contextmenu(撤去) / touch (preventDefault でクリック発火抑止)
- 効果音: Web Audio API オシレータ(サイコロ大戦争と同方式)。短いクールダウン(90ms)で大量同時発火を抑制
- 永続化: `localStorage["kgames-lemmings"]` に `cleared`(各ステージのグレード)/ `stage8Grade`(隠し解放判定用)/ `muted` を保存
- ビルド工程なし。`index.html` を編集して push するだけで反映 (1〜2分)

### 画面構成

│ Stage 1: はじめの一歩  🚶 0/30  🏆 0/18 │  HUD (上段)
│   💀 0/6  ⏱ 2:00     [⏸][⟳][🔊][×]    │
├────────────────────────────────────────┤
│                                          │
│         背景グラデ + ピラミッド          │
│   [🚪]→ メジェド  → →  [▲ ピラミッド]    │  Canvas
│   ===地面===  穴==棘=== 地面=====        │  (960×544)
│                                          │
├────────────────────────────────────────┤

タイトル画面はメジェド6体が並んでバウンス、メニュー(はじめる/ステージ選択/操作説明)+ ミュート + 戻るリンク。
操作説明画面は **HTML4風レトロスタイル**(白背景/MS UI Gothic/★見出し/`border="1"` テーブル)で別レシピを採用 → `k-books/引き継ぎ書_HTML4風スタイル.md` 参照。

### コアメカニクス

#### メジェドの状態機械

| 状態 | 説明 |
|---|---|
| `walk` | 右(または左)へ等速で歩行。`speed * BASE_SPEED * speedModifier` |
| `fall` | 自由落下(`GRAVITY = 240 px/s²`)。落下安全効果中は終端速度 50px/s で安全 |
| `dead` | 棘・水・溶岩への接触/落差死/画面外落下で確定。死亡カウント++ |
| `exit` | ゴール到達。`saved` カウント++、ファラオなら `savedPharaoh` も |

#### 衝突解析(タイル列を跨ぐタイミングで判定)

進行先の列 `aheadCol` について、`bodyAhead` (footRow-1) と `headAhead` (footRow-2) と `floorAhead` (footRow) を読み取り:

1. **範囲外 (画面端)** → 反転
2. **bodyAhead が SOLID**:
   - `T.LADDER` → ladder 列の最上段までテレポート登攀
   - `headAhead も SOLID` → 2マス壁で反転
   - それ以外 → 1マス段差として自動上り(footRow を 1 減らす)
3. **floorAhead が HAZARD** → その場で hazard 死
4. **floorAhead が空** → 落下開始
5. それ以外 → 通常歩行で px 加算

#### キャラ別の挙動差

| ID | 名前 | 速度修飾 | 特徴 |
|---|---|---|---|
| A | 基本 | ×1.0 | 標準。最も多く出現 |
| B | ファラオ | ×0.9 | 黄金ハロ表示。**stage 9 のみ混入(終盤5体固定)・全員救出必須・1体でも死ぬと即敗北** |
| C | 戦士 | ×1.0 | 仕様上は A と同じ挙動。槍を持つ案内役(タイトル等で活用余地) |
| D | 司祭 | ×0.7 | スロー歩行。リズム崩しの要素 |
| E | ミイラ | ×0.85 | **時々呪いで反転**(`walk` 中に約 3.5 秒以上経過 + 2% 確率で `dir` 反転)。stage 9 のみ混入 |
| F | 子供 | ×1.4 | 小柄・速い。中盤以降に混入 |

#### アイテム7種 (PLACED_TILES)

| ID | アイコン | 占有 | 効果 |
|---|---|---|---|
| BLOCK | ⬛ | 1×1 | SOLID。万能足場(穴埋め・棘の上覆い・かさ上げ) |
| RAMP | 🔺 | 1×1 | SOLID。視覚的に三角だが物理的には BLOCK と同じ |
| BRIDGE | 🌉 | 3×1 | SOLID 3連続。広い穴を1アイテムで橋渡し |
| LADDER | 🪜 | 1×3 | SOLID 縦3連続。ぶつかると **上端までテレポート登攀** |
| ARROW_R | ➡ | 1×1 | 通過時 `dir = 1` 強制(右向き)。SOLID ではなく通過可能 |
| ARROW_L | ⬅ | 1×1 | 通過時 `dir = -1` 強制(左向き)。SOLID ではなく通過可能 |
| PARACHUTE | 🍃 | 1×1 | 通過時 `hasParachute = true`。以後の落下が安全に(終端速度50px/s 化)。右肩に小さな葉アイコン表示 |

配置: アイテムボタン選択 → 空セルをタップ/クリック。撤去: アイテム未選択でセルタップ、または PC では右クリック。BRIDGE/LADDER は1アイテム=3セット同時配置/撤去。

### ステージ仕様

| # | 名前 | 出現 | 必要 | 許容死 | 速度 | 出現間隔 | 制限時間 | 特徴 |
|---|---|---|---|---|---|---|---|---|
| 1 | はじめの一歩 | 30 | 18 | 6 | 1.0× | 1.4s | 120s | 平地のみ・チュートリアル |
| 2 | 落とし穴 | 40 | 26 | 8 | 1.1× | 1.2s | 120s | 2マス幅の穴1個 |
| 3 | 棘の床 | 50 | 32 | 10 | 1.2× | 1.0s | 130s | 3マス棘+橋orブロックで覆う |
| 4 | 段差を越えて | 60 | 38 | 10 | 1.3× | 0.85s | 140s | 2×2 高壁。F(子供) 混入開始 |
| 5 | 砂漠の罠 | 70 | 44 | 12 | 1.5× | 0.7s | 150s | 4幅穴+棘ゾーン。D(司祭) 混入 |
| 6 | 崖からの大跳躍 | 80 | 50 | 14 | 1.7× | 0.55s | 160s | 高低差6マスの崖。落下安全必須 |
| 7 | 試練の道 | 90 | 58 | 14 | 1.9× | 0.45s | 170s | 穴2+高壁+棘の複合 |
| 8 | メジェドの王国 | 100 | 70 | 15 | 2.2× | 0.35s | 180s | 棘群+大穴+高台+水。総合試験 |

#### クリア/失敗判定 (`checkEndCondition`)

- **失敗即時**: 死亡数 > `stage.fail` / 隠しステージでファラオ死 (`deadPharaoh > 0`)
- **タイムアップ**: `timeLeft <= 0` でその時点の救出数で判定
- **通常**: 全メジェドが ゴール または 死亡 で決着 (`allSpawned && remainingActive() === 0`)
- **手動**: ポーズメニューの「🏁 いまの結果で評価」で打ち切り(規定数を満たしていれば勝利)

#### 評価 (S / A / B / C)

- `ratio = saved / total` の救出率と死亡数で判定:
  - **S**: 死亡 0 + ratio ≥ 0.9
  - **A**: 死亡 ≤ floor(fail × 0.4) + ratio ≥ 0.75
  - **B**: ratio ≥ 0.65
  - **C**: 規定数を満たすが ratio < 0.65
- **stage 9 解放条件**: stage 8 を S または A でクリア (`State.progress.stage8Grade in ['S','A']`)

## 既知の制約 / 今後の余地

### 既知の制約

- **ディレクトリ名が大文字始まり (`Lemmings/`)**: 他ゲームと揃わないが既存QR/URL を維持するため放置。改名する場合 `git mv Lemmings lemmings` + QR再焼成 + 引き継ぎ書修正が必要
- **詰みパターン**: ブロックを誤って配置しメジェドが2つの壁の間で永久反転するとアクティブが0にならない。**制限時間で強制終了**して救済しているが、視覚的に「詰んだ」を伝えるUIはない
- **アイテム配置の精度**: タッチ環境ではマス境界のタップが難しい場面あり。ホバープレビュー(緑/赤枠)で配置可否は出すが、誤配置 → 撤去の往復は発生しうる
- **ステージバランス未調整**: 各ステージの予算は理論上クリア可能なラインで設定したが、実プレイで「易しすぎ/詰みやすい」配点が判明する可能性あり
- **C(戦士) の活躍場面が薄い**: 仕様上は A と同等。タイトル画面の意匠役のみ。今後ステージにメカニクスを追加する余地あり

### 今後の余地

- **スポーン途中の中断ボタン**(全スポーン待たず終了させる) — 現状ポーズの「🏁 いまの結果で評価」で代替済み
- **アイテム割り当てのドラッグ移動**: 現状は配置→撤去→再配置の2ステップ。長押しドラッグで再配置できると操作感UP
- **ステージエディタ**: 自作ステージを localStorage に保存。`STAGES` 配列の構造をJSON化すれば外部入出力もしやすい
- **チャレンジモード**: 全ステージ S 評価コンプリート報酬・タイムアタック等
- **音楽 BGM**: 効果音のみで BGM がない。Web Audio で簡易シンセを足すのも案
