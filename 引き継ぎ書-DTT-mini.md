## 概要

2007年のFlashゲーム **Desktop Tower Defense** のシンプル復刻版。
タワーで迷路を作って敵を誘導するオープンフィールド型のタワーディフェンス。
スコアアタック型（クリア条件無し、何ウェーブまで耐えられるかの挑戦）。

## ファイル

- `index.html` … PC（マウス操作）版。サイドパネルUI。
- `tablet.html` … iPad/タブレット（タッチ操作）版。縦レイアウト・タップで売却・強化。
- `guide.html` … Win98 風レトロUI の攻略ガイド（外部リソース：`98.css` のみ）
- `98.css` … jdan/98.css v0.1.21（exact pinned、ローカル配置）
- ゲーム本体（`index.html` / `tablet.html`）は単一HTMLで自己完結（サーバー不要）
- ガイドは `guide.html` ＋ `98.css` の2ファイル一組

両ゲームファイル（index/tablet）はゲームロジックがほぼ同一。仕様変更時は両方に反映する必要がある。

## 起動方法

- PC: `index.html` をダブルクリックでブラウザ表示。
- iPad: AirDrop / iCloud Drive 等で `tablet.html` を転送 → ファイルアプリから Safari で開く。
  - PC立ち上げ不要・ネット接続不要。
  - ホーム画面に追加すれば全画面アプリ風に動く。

## 操作方法

### PC (index.html)
- **左クリック（空きマス）**: 選択中タワー種を設置
- **左クリック（タワー）**: 選択（黄枠＋射程円表示、右パネルに強化/売却ボタン）
- **右クリック（タワー）**: 即時売却（70%返金）
- 右パネル「強化」ボタン: 1段階アップグレード
- 右パネル「売却」ボタン: 70%返金

### iPad (tablet.html)
- **タップ（空きマス）**: 選択中タワー種を設置
- **タップ（タワー）**: 選択
- 下バー「強化」: 1段階アップグレード
- 下バー「売却」: 70%返金

## ゲーム仕様

### マップ
- 20×15 グリッド（CELL=30px の論理単位）
- 左中央 (0,7) = IN（緑）、右中央 (19,7) = OUT（茶）
- 経路を完全にふさぐ配置は禁止（飛行敵は経路無視のため例外）

### 初期値
- お金: $75 / ライフ: 20
- 第1ウェーブまで20秒、ウェーブ間10秒（自動開始、ボタンで早送り可）

### タワー（3段階強化）

**Lv1〜3（最初から使用可能）:**

| **大砲** (範囲) | $30 / 攻14 / 射105 / 1.10s / 範囲38 | +$55 / 攻34 / 射122 / 範囲48 | +$120 / 攻80 / 射142 / 範囲60 |
| **対空** (飛行のみ) | $25 / 攻18 / 射110 / 0.40s | +$40 / 攻38 / 射125 / 0.35s | +$90 / 攻75 / 射140 / 0.30s |

|---|---|---|---|---|
| **4** | W30 | $100 / 攻70 / 0.18s / 射110 | $280 / 攻220 / 0.75s / 範囲72 | $220 / 攻180 / 0.27s / 射155 |
| **5** | W40 | $280 / 攻220 / 0.15s / 射120 | $700 / 攻600 / 0.65s / 範囲85 | $600 / 攻550 / 0.24s / 射170 |
| **6** | W50 | $700 / 攻700 / 0.12s / 射130 | $1800 / 攻1700 / 0.55s / 範囲100 | $1500 / 攻1600 / 0.21s / 射185 |

- 売却額 = 累計投資額 × 0.7（切り捨て）
- レベル表示: 左上の小さな点（Lv1〜3=白、Lv4〜5=金、Lv6〜7=赤）
- 大砲は範囲攻撃（splash）
- 対空は `antiAir: true` フラグで地上敵を完全無視（射程内でも撃たない）
- Lv4以降は `unlockWave` フラグで強化を制限。未解禁時は強化ボタンが「Wave XX で解禁」表示

### 敵

| 種類 | HP | 速度 | 報酬 | 備考 |
|---|---|---|---|---|
| basic | 22 | 50 | 3 | 通常 |
| fast | 14 | 95 | 4 | wave≥3 から |
| flying | 32 | 70 | 6 | wave≥4。**経路無視で直線飛行**（ひし形＋影） |
| tank | 110 | 28 | 9 | wave≥5。HP高 |
| bonus | 44 | 45 | 15 | **wave % 5 === 0 で各ウェーブ末尾に1体**。basic比 HP 2倍・報酬 5倍。金色＋ハロー＋$マーク |

### スケーリング
- HP: `1 + (wave-1) * 0.18`、wave>20 で `* 1.12^(wave-20)` の複利
- 速度: wave>20 で `1 + (wave-20) * 0.025`（最大1.6倍）
- 報酬: wave>20 で 4ウェーブごとに +$1
- 出現数: `6 + wave * 2`、出現間隔は wave とともに短縮（最短0.35s）
- ウェーブクリアボーナス: $20 + wave×5

### 早期召喚
- 進行中でも「次のウェーブを召喚」ボタンが押せる
- 早期召喚ボーナス: +$(5 + wave×2)
- 残党と新規が同時にマップ上に存在しうる
- **早期召喚時は残キューを先に排出（保存wave値でHP計算）→ 新wave分も即時バースト**。`spawnQueue` 要素に `wave` を保存、`spawnEnemy(type, w)` の第2引数で wave を指定可能。タイマー満了の通常開始は従来どおり順次召喚（こちらも `wave` がキューに保存される）。これにより、連打しても／自動開始の途中で早期召喚しても、各敵はキューに入った瞬間の wave のHP/速度/報酬で固定される
- ボタン状態別の挙動：
  - `▶ ウェーブ開始` / `▶ Xs (今すぐ)`（`inWave=false`）→ 通常開始（順次スポーン）
  - `▶ WX 召喚`（`inWave=true`）→ 早期召喚（バーストスポーン）
  - 自動タイマー満了 → 通常開始（順次スポーン）

### 裏技（隠し要素）
- **四隅対空ボーナス**: マップの4隅 (0,0), (19,0), (0,14), (19,14) に対空タワーをすべてLv2以上で配置すると、$9999 を獲得
- 1ゲーム1回限り（`cornerBonusTriggered` フラグで管理、`saveGame` の `cornerBonus` フィールドにも保存）
- `checkCornerBonus()` がタワー配置時と強化時に呼ばれて発動条件をチェック
- リスタートでフラグはリセット
- セーブ形式は v1 から v2 に上げた（v1 も読み込み可、`cornerBonus` は未定義なら false 扱い）
- ゲームUI上にはヒントを置かず、`guide.html` の「RUMOR.DAT」セクションで2008年BBS風に匂わせる演出のみ

## 攻略ガイド（guide.html）
- **Win98 風レトロUI**：jdan/98.css v0.1.21 をローカル配置、デスクトップ色 `#008080`
- **フォント**：`MS UI Gothic` 優先、`MS Gothic` をフォールバック・等幅／数字用に併用。`-webkit-font-smoothing: none` でビットマップ感を強化
- **半角数字対策**：`table.num`・`code`・`pre.ascii` で `monospace` フォールバックが Courier 等に落ちるのを防ぐため、`MS Gothic` を `!important` で強制（`font-variant-numeric: tabular-nums`）
- **構成**：概要／戦略の核／ビルド進行／タワー比較／敵キャラ／マップレイアウト／注意点／推定到達ライン／RUMOR.DAT
- **RUMOR.DAT**: 黒×緑のCRT端末風 `pre.ascii` で2008年掲示板ログ風の都市伝説を表示（裏技のヒント）。具体的な座標・タワー名・Lv数値・$額は出さず、「隅っこ」「空飛ぶやつ撃つ用」「Lv2 だった気がする」「$9999 とかいう数字」と段階的に匂わせる
- **ヘルプリンク**: 両ゲーム画面に `#help-link`（24×24px の半透明グレー丸 `?`）を `target="_blank"` で配置。tablet は topbar 内（mute と wave-btn の間）、PC はサイドパネル下

## スマホ縦レイアウトのキャンバスジャンプ対策（tablet.html）
- 過去問題：`#topbar { flex-wrap: wrap }` で `#wave-btn` のテキスト変動（「ウェーブ開始」⇄「次まで Xs (今すぐ)」⇄「次のウェーブ召喚 (W2)」）に応じて折返しが発生 → `#canvas-wrap` の高さが変わり `fitCanvas()` が再計算してマップが上下にジャンプ
- 修正内容：
  - `#topbar { flex-wrap: nowrap; min-height: 44px }`（≤480px は 40px）で折返し禁止＋高さ確保
  - `.stat` / `#mute-btn` / `#help-link` に `flex-shrink: 0`（縮まない）
  - `#wave-btn` は `min-width: 0; flex: 0 1 auto; text-overflow: ellipsis`（残スペースに合わせて縮む）
  - ボタンテキストをコンパクト化：`▶ ウェーブ開始` / `▶ Xs (今すぐ)` / `▶ WX 召喚`

## 実装メモ

### 全画面表示
- `fitCanvas()` でビューポートに合わせて canvas 内部解像度を `display × DPR` で設定
- `ctx.setTransform(scale, ...)` で論理座標（CELL=30）のまま描画
- `resize` / `orientationchange` で再計算

### 経路探索
- BFS（`findPath`）。タワー設置時に `canPlaceTower` で
  - START→GOAL の経路存在チェック
  - 各非飛行敵の現在位置→GOAL の経路存在チェック
- 設置／売却で `recomputePaths()` を呼び全敵の経路を再計算（飛行敵はスキップ）

### ターゲット選択
- タワーは射程内の敵のうち**ゴールに最も近い**敵を狙う（地上・飛行を統一して扱える）

### 飛行敵の判定
- `flying: true` フラグ。`path` の代わりに `goalX/goalY` への直線移動
- ターゲット選択・描画・経路再計算で個別分岐

## 既知の制限・今後の候補

- **リスタート機能あり**: ゲームオーバー後にHTMLオーバーレイで「もう一度プレイ」ボタンを表示し、`restartGame()` で全状態をリセット。
- **ボス／クリア条件なし**: スコアアタック型として割り切り。
- **タワーは3種類**: スロー・ブースト・地上特化など追加余地あり。
- **マップは固定 20×15、IN/OUTも固定位置**: 複数マップ・難易度モードの余地あり。
- **セーブデータあり**: ウェーブ間に `localStorage` 自動セーブ（key `dtt-save`）。ハイスコアは top5 を `dtt-hiscores` に保存。
- **発射音のみ**: 命中音／ウェーブ開始SE／BGMは未実装。
- **両ファイルでロジック重複**: 共通JSファイルに切り出すと保守は楽になる（ただし「単一HTMLで配布」のメリットは失われる）。
