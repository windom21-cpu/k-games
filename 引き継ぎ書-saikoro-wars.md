# 引き継ぎ書 — サイコロ大戦争

最終更新: 2026-05-05 / 現行バージョン: v1.2.0

---

## 概要

2001年頃に SDIN(Gamedesign)から公開された Web ゲーム **Dice Wars**(陣地+各マスにサイコロ → 隣接マスに攻撃 → 双方の合計で勝負 → ターン終了で最大連結領域分の補充)の簡略再実装。

題材として **関東地方の7都県**(群馬・栃木・茨城・埼玉・東京・千葉・神奈川)を六角タイル46マスで表現。各マスは1都県に属するが、ゲーム上は属性として表示されているだけでメカニクス上の意味は持たない(視覚的な雰囲気付けのみ)。

公開URL: https://windom21-cpu.github.io/k-games/saikoro-wars/
QRコード: `saikoro-wars/qr.png`

## 技術スタック

- 純粋な HTML5 + CSS + JavaScript (バニラ JS)
- 描画: SVG (動的に DOM 構築、毎フレーム再生成)
- 入力: click イベント(SVG polygon に `data-id`、SVGルートに event delegation)
- レイアウト: CSS grid + flex、`viewBox` で SVG をレスポンシブ
- 効果音: Web Audio API(オシレータベースの軽量ビープ)
- 永続化: ミュート設定のみ localStorage(`saikoro-wars-mute`)。ゲーム状態は永続化なし(リロードで初期化)

各マスの内側要素(上から):
1. 都県名の1文字(群/栃/茨/埼/東/千/神)
2. サイコロ数の数字(中央、大きく)
3. 8段のピップバー(白=詰まってる、暗=空きスロット)。`SEG_COUNT` で本数定義(現状 8 = 1マス上限と一致)

### マップ — 関東風hex(46マス)

`HEX_DATA` に `[col, row, prefCode]` の配列で定義。pointy-top の odd-r オフセット座標。

都県別マス数(配置はおおむね実際の関東地方を抽象化):

| 都県 | コード | マス数 |
|---|---|---|
| 群馬 | GU | 6 |
| 栃木 | TO | 6 |
| 茨城 | IB | 7 |
| 埼玉 | SA | 7 |
| 東京 | TK | 6 |
| 千葉 | CH | 8 |
| 神奈川 | KN | 6 |
| **合計** | | **46** |

⚠ 隣接関係は六角形の幾何学に従うため、現実の県境とは一部異なる(例: 神奈川と千葉が直接隣接する位置がある)。これはゲーム性のため許容。

### コアメカニクス

#### 初期配置
- 4つの「最遠点」種マスを max-min 距離で選び、各プレイヤーに割り当て
- ラウンドロビンで隣接の未占領マスを1つずつ展開(クラスタ状の領地ができる)
- 各マスに最初1個サイコロを置き、各プレイヤーに領地数×2個ぶん追加で振り分け(満タン=8で打ち止め)

→ 各プレイヤーは約11〜12マス・合計サイコロ約34個でスタート。

#### 攻撃
1. 自分のマス(2個以上のサイコロ)→ 隣接の他プレイヤーマスをタップで攻撃
2. 双方の **全サイコロを振って合計を比較**
3. 攻撃側合計 > 守備側合計 → 攻撃成功(同点は守備側勝ち)
   - 守備マスを攻撃側の所有に変更
   - 攻撃マスのサイコロは1個残し、残り(攻撃マス元数 − 1)個を奪った先へ移動
4. 攻撃側合計 ≦ 守備側合計 → 失敗
   - 攻撃マスのサイコロが1個まで減る(守備マスは無傷)

##### 戦闘演出(v1.2.0〜)
- `rollDice(n)` で個別の出目配列を保持(`G.battle.adArr` / `ddArr`)
- 攻撃元 → 守備マス への **黄色い矢印**(影付き)を SVG `<line>` + 三角ヘッドの `<polygon>` で描画
- 各マスの上に **サイコロ目バッジ**: 攻撃側は黄枠、守備側は赤枠で `⚀⚄⚂=合計` 形式表示
- `setTimeout` 600ms で勝敗音(`SFX.hit` / `SFX.miss`)、950ms で盤面更新

#### ターン終了 → 補充
- 自分の **最大連結領域サイズ** ぶん + 備蓄(stock)ぶん のサイコロを取得
- 自分の領地のうちサイコロ8個未満のマスにランダムに1個ずつ配置
- 全マス満タンで配置できなかった分は備蓄(最大64個まで保持)

#### ターン進行
- 0(人)→ 1(CPU青)→ 2(CPU緑)→ 3(CPU黄)→ 0... の固定順
- 脱落したプレイヤー(領地0)はスキップ

### CPU AI(3段階)

| 難度 | アルゴリズム |
|---|---|
| **よわい** | 攻撃側 ≥ 守備側 の手のみ採用、ランダム選択(無策・無計画) |
| **ふつう** | 攻撃側 > 守備側 を優先、`攻 − 守` の差が大きい上位3手からランダム |
| **つよい** | 期待勝率(`WIN_PROB[a][d]`)+ 領土拡大ボーナス(連結領域成長 × 25)− 反撃リスク(占領後の隣接敵サイコロ合計 × 0.5)で評価。勝率55%未満は除外、最高得点が30未満ならターン終了 |

`WIN_PROB[a][d]` は a 個と d 個のサイコロ合計分布を畳み込んで攻撃側の勝率を厳密計算したテーブル(起動時に1回計算)。

### 効果音(v1.2.0〜)

Web Audio API ベースの軽量SFX(オシレータでビープを生成、外部音源なし)。`SFX` IIFE オブジェクトに集約。

| メソッド | タイミング | 音 |
|---|---|---|
| `SFX.diceRoll()` | 攻撃時 / PKラウンド開始時 | 7発のカチャカチャ系(160-300Hz triangle) |
| `SFX.hit()` | 攻撃成功時 | 上昇2音(C5→G5 square) |
| `SFX.miss()` | 攻撃失敗時 | 下降2音(E4→G3 sawtooth) |
| `SFX.select()` | 自分マス選択時 | 短い高音(900Hz square) |
| `SFX.turnEnd()` | ターン終了押下時 | 中低音(280Hz sine) |
| `SFX.pkRoundWin/Lose()` | PKラウンド勝敗時 | 短い上昇/下降2音 |
| `SFX.win()` / `SFX.lose()` | 結果モーダル表示時 | 4音アルペジオ |

ミュートトグル: topbar の 🔊/🔇 ボタン。状態は `localStorage('saikoro-wars-mute')` に永続化。
AudioContext は autoplay 制限のため最初のユーザー操作(`pointerdown`/`keydown`)で `init` する。

### 終了条件

- 1人だけ残れば勝利
- **人(プレイヤー0)が脱落した時点で即終了**(CPU同士の続きは見せない)
- 結果モーダルで勝者表示 → 「もう一度」(同難度で再戦)/ 「タイトルへ」

### 残り2人時の特殊フロー(v1.1.0〜)

「残り2人になると毎ターン全マスがサイコロ満タンに溜まって消化試合化する」問題への対策。
攻撃で誰かが脱落した瞬間(`checkAlive`)、生存数 == 2 になったらフラグ `endgameTriggered` を立て、
領地差を見て次のいずれかに分岐する。

#### (A) 拮抗 → 決戦モード(サイコロPK)

- **発火条件**: `|あなたの領地数 − 相手の領地数| ≤ ENDGAME_TIE_DIFF`(初期値 6マス)
- 通常ターンを停止して `pk-modal` をフルスクリーン表示
- **5本3勝先取**(`PK_BEST_OF`=5、`PK_WIN_THRESHOLD` = ⌈5/2⌉ = 3)
- 各ラウンド双方が **3個ずつサイコロを振って合計勝負**(`PK_DICE_PER_SIDE`=3)
- 同点ラウンドはノーカウントでやり直し(round 番号は据え置き)
- ロール演出 ~700ms(`setInterval` でユニコードのサイコロ目 ⚀⚁⚂⚃⚄⚅ をランダム表示)→ 確定値 → 合計表示 → 結果(勝ち/負け/引き分け)→ スコア更新
- 勝者決定で `finalizeFromPK()` → `alive[]` を更新して `showResult()`
- 中断不可だが `投了してタイトルへ` ボタンでタイトルへ戻れる

#### (B) 圧倒/されている → 服従選択モーダル

- **発火条件**: 拮抗判定に該当しなかった残り2人状態
- `submit-modal` を表示。あなたが優勢か劣勢かでタイトル文と yes ラベルを変える
  - 優勢: 「[相手] を服従させますか?」 / `はい(服従させる)` / `いいえ(続ける)`
  - 劣勢: 「[相手] に服従しますか?」 / `はい(服従する)` / `いいえ(続ける)`
- **はい** → 該当者を `alive=false` にして `showResult()`
- **いいえ** → `endgameDeclined=true` を立てて通常戦に復帰。以降は再表示しない(投了ボタンで抜けられる)
  - AIの番だった場合は中断していた `aiTurn` を再開する

### 入力 / UX

- 自分のマスをタップ → 選択(黄色枠)、攻撃可能な敵マスに白点線パルス
- 自分のマス再タップ or 他の自分マスをタップで選択切替/解除
- 敵マス(攻撃可能)タップ → 攻撃実行、両マスの上にロール結果(黄色バッジ)を約950ms表示してから盤面更新
- 「ターン終了」ボタン押下で補充→次プレイヤーへ
- 「投了してタイトルへ」(confirm 経由)でタイトル復帰

## 既知の制約 / 今後の余地

- **マップ形状は固定**(関東のみ)。`HEX_DATA` を差し替えれば別地方も作れるが、隣接性と総マス数のバランスは要設計
- **3人/2人/5人プレイは未対応**。プレイヤー数を可変にするなら色・名前・初期配置・ターン進行を整理する必要あり
- **戦闘演出はサイコロ目を出すが、振動アニメは無し**(v1.2.0で出目は表示するようになった。PKモードのような連続再ロール演出は通常戦には入れていない)
- **盤面シャッフル開始**: 初期配置はクラスタ展開のため、稀に1人が極端に有利な開始になることがある。完全均等にしたいなら配置後のリバランスを足す
- **保存機能なし**: リロードで初期化。途中保存(localStorage)は v1.x で要望次第
- **endgame は残り2人になった瞬間に1度だけ判定**。判定後に状況が変動しても再判定しない(設計判断)。`endgameDeclined` で通常戦に戻った場合、その後さらに戦況が拮抗/圧倒に振れても再表示しない
